## Q3: 의무담보 최소화 추천

### 고객 질문
> "의무담보를 최소화한 상품을 추천해줘. 기본 계약만으로 구성된 저렴한 상품을 찾고 싶어."

### 현재 상태
❌ **제공 불가**

### 이유

보험 상품의 **의무담보(기본 계약)**와 **선택담보(특약)**를 구분하는 정보가
현재 시스템에 등록되어 있지 않습니다.

**구체적인 문제**:

1. **담보별 필수/선택 구분 불가**
   - 약관 문서에는 담보별 조건이 상세히 기재되어 있음
   - 하지만 "이 담보가 필수인지, 선택 가능한지"에 대한 명시적 표시가 부족

2. **상품 구조 분석 미구현**
   - 현재 시스템은 담보별 조건 추출에 집중
   - 상품 전체 구조 (주계약/특약 계층) 분석 기능이 없음

3. **데이터 신뢰도**
   - 8개 보험사 모두 의무담보 정보 확인 불가 (0%)
   - "데이터 없음" 상태

### 잘못된 정보 제공 예시 (제공하지 않는 이유)

❌ **잘못된 응답** (시스템이 추측할 경우):
```
삼성화재: A1300(상해사망)이 의무담보
→ 실제로는 CI보험에서는 선택담보일 수 있음
→ 고객이 "필수"로 오해하고 불필요한 비용 지불
```

✅ **올바른 태도**:
```
삼성화재: 의무담보 구분 정보 없음
→ 고객이 직접 약관 확인 또는 "기본형" 상품 문의
```

### 필요한 시스템 개선

**기술적 보완 사항**:
1. 보험사별 상품 구조 분석 (주계약/특약 구분)
2. 의무담보 목록 추출 로직 개발
3. 담보별 필수/선택 속성 SSOT 구축

**예상 소요 시간**: 5-7일 (새로운 파이프라인 단계 필요)

### 현재 이용 가능한 대안

#### 방법 1: 보험사 "간편 가입" 상품 조회
각 보험사 홈페이지에서 **기본형** 또는 **간편형** 상품 검색
- 삼성화재: www.samsungfire.com (상품 검색 → "간편형")
- 메리츠화재: www.meritzfire.com (상품 비교 → "기본형")

**특징**:
- 보험사가 미리 구성한 "최소 구성" 상품
- 의무담보만으로 구성된 경우가 많음
- 보험료가 상대적으로 저렴

#### 방법 2: 설계사에게 "최소 구성" 견적 요청
전화/방문 상담 시 **반드시 요청해야 할 사항**:
1. "의무담보만으로 구성된 최저 보험료 견적을 받고 싶습니다."
2. "어떤 담보가 필수이고, 어떤 담보가 선택인지 알려주세요."
3. "선택담보를 제외한 보험료는 얼마인가요?"
4. "약관에서 주계약/특약 구분을 확인할 수 있는 페이지를 알려주세요."

#### 방법 3: 비교 사이트 "특약 제외" 옵션 사용
일부 보험 비교 사이트에서 제공하는 기능:
- "기본 보장만" 필터
- "특약 제외" 옵션
- "최저 보험료" 정렬

**주의사항**: 사이트마다 "기본 보장" 정의가 다를 수 있으므로 반드시 담보 목록 확인

## Q11: 암직접입원비 보장한도 비교

### 고객 질문
> "암직접입원비 담보 중 보장한도가 다른 상품을 비교해줘."

### 현재 상태
❌ **제공 불가**

### 이유

암직접입원비는 **"하루 입원당 2만원"** 같은 **일당 금액**과
**"연간 최대 90일"** 같은 **기간 한도**가 모두 중요합니다.

현재 시스템은:
- ✅ 일당 금액 정보: 확보됨 (대부분 2만원으로 동일)
- ❌ 기간 한도 정보: 미확보 (약관 문서에는 존재하나 추출 실패)

**구체적인 문제**:

1. **데이터 구조 불일치**
   - 보험 데이터 저장 방식이 "한도" 질문에 최적화되지 않음
   - "일당 금액"과 "기간 한도"가 구분되지 않음

2. **추출 실패**
   - 약관에 "90일한도", "120일한도" 문구는 존재
   - 하지만 여러 담보의 한도가 혼재된 페이지에서 정확한 추출 불가

3. **데이터 신뢰도**
   - 7개 보험사 중 일당 금액만 확보 (85.7%)
   - 기간 한도는 0개사 확보 (0%)

### 잘못된 정보 제공 예시 (제공하지 않는 이유)

❌ **잘못된 응답** (시스템이 추측할 경우):
```
삼성화재: 일당 2만원 (한도 90일)
→ 실제로는 "암 요양병원 입원일당"의 한도였을 가능성
→ 고객이 일반 병원 입원도 90일 한도로 오해
```

✅ **올바른 태도**:
```
삼성화재: 일당 금액 확인 가능, 기간 한도 확인 불가
→ 고객이 약관에서 "암직접치료입원일당" 특약의 한도 직접 확인
```

### 필요한 시스템 개선

**기술적 보완 사항**:
1. 데이터 저장 구조 개선 (일당 금액 vs 기간 한도 분리)
2. 담보별 한도 추출 로직 강화
3. 슬롯 스키마 재설계 (금액/한도 구분)

**예상 소요 시간**: 3-5일 (슬롯 스키마 재설계 필요)

### 현재 이용 가능한 대안

#### 방법 1: 보험사 상담 센터 문의
전화 문의 시 **반드시 확인해야 할 질문**:
1. "암직접입원비(또는 암입원일당) 담보의 일당 금액이 얼마인가요?"
2. "연간 또는 보험기간 중 최대 몇 일까지 지급되나요?"
3. "요양병원과 일반 병원의 한도가 다른가요?"
4. "약관 몇 페이지에 한도가 명시되어 있나요?"

#### 방법 2: 약관 원문 직접 확인
각 보험사 홈페이지에서 **상품 약관** 다운로드 후 확인
- 삼성화재: www.samsungfire.com
- 메리츠화재: www.meritzfire.com

**확인 방법**:
1. 약관 PDF 다운로드
2. "암직접치료입원일당" 또는 "암입원일당" 특별약관 찾기
3. "보장내용" 표에서 "지급한도" 또는 "지급일수" 항목 확인
4. "연간 N일", "보험기간 중 N일" 같은 표현 찾기

#### 방법 3: 보험 설계사에게 설계서 요청
설계서에는 담보별 세부 조건이 표시되므로:
- "암직접입원비 일당 금액"과 "연간 한도(일수)"를 동시에 확인 가능
- 여러 보험사 설계서를 나란히 비교

## Q5: 감액/면책 기간 상세 비교

### 고객 질문
> "암 진단 시 90일 면책 직후 100% 지급 vs 1년/2년 감액(50%) 존재 상품을 구분해줘."

### 현재 상태
❌ **제공 불가**

### 이유

보험 약관에서 **면책기간**과 **감액 조건**에 대한 문구를 발견했으나,
해당 조건이 **암진단비 담보에만 적용되는지 확인할 수 없습니다**.

**구체적인 문제**:

1. **약관 문서 구조**
   - 한 페이지에 여러 담보의 조건이 혼재되어 있음
   - 예: "면책기간 90일" 문구가 암진단비, 암입원비, 암수술비 중 어디에 적용되는지 불명확

2. **담보 귀속 판정 한계**
   - 현재 시스템은 "이 조건이 어느 담보의 것인지" 확인하는 기준이 불충분
   - 잘못된 정보를 제공할 위험이 있어 의도적으로 제공하지 않음

3. **데이터 신뢰도**
   - 8개 보험사 중 1개만 확실한 데이터 확보 (12.5%)
   - 나머지 7개사는 "데이터 없음" 상태

### 잘못된 정보 제공 예시 (제공하지 않는 이유)

❌ **잘못된 응답** (시스템이 추측할 경우):
```
삼성화재: 90일 면책 후 100% 지급
→ 실제로는 "암입원비"의 조건이었을 가능성
→ 고객이 암진단비로 오해하고 가입 결정
```

✅ **올바른 태도**:
```
삼성화재: 데이터 부족으로 확인 불가
→ 고객이 직접 약관 확인 또는 상담 센터 문의
```

### 필요한 시스템 개선

**기술적 보완 사항**:
1. 약관 문서 내 담보별 구조화 분석 강화
2. 면책/감액 조건의 담보 귀속 확인 로직 보완
3. 문맥 분석 기반 조건-담보 연결 알고리즘 개발

**예상 소요 시간**: 2-3일 (전체 보험사 재분석 필요)

### 현재 이용 가능한 대안

#### 방법 1: 약관 원문 직접 확인
각 보험사 홈페이지에서 **상품 약관** 다운로드 후 확인
- 삼성화재: www.samsungfire.com
- 메리츠화재: www.meritzfire.com
- (기타 보험사 홈페이지)

**확인 방법**:
1. 약관 PDF 다운로드
2. "암 진단비" 특별약관 찾기
3. "보장개시일" 또는 "면책기간" 항목 확인
4. "감액 지급" 또는 "지급률" 항목 확인

#### 방법 2: 보험사 상담 센터 문의
전화 문의 시 **반드시 확인해야 할 질문**:
1. "암진단비 담보의 면책기간이 몇 일인가요?"
2. "면책기간 이후 바로 100% 지급되나요?"
3. "감액 지급 기간이 있나요? (예: 1년 50%, 2년 50%)"
4. "감액이 없다면, 약관 몇 페이지에 명시되어 있나요?"

#### 방법 3: 보험 설계사 상담
설계사에게 **설계서 + 약관**을 함께 요청하여 직접 확인

## 서비스 철학

### 왜 "추측"하지 않나요?

**원칙**: **확인할 수 없는 정보는 제공하지 않습니다.**

1. **고객 보호**
   - 잘못된 정보로 인한 가입 결정 방지
   - 분쟁 발생 시 책임 소재 명확화

2. **신뢰성 우선**
   - 80% 정확한 정보보다 100% 확실한 정보만 제공
   - "모르는 것은 모른다"고 명시

3. **투명성**
   - 시스템 한계를 숨기지 않음
   - 고객이 대안을 선택할 수 있도록 안내

### "데이터 부족" vs "확인 불가" 차이

| 상황 | 의미 | 고객 행동 |
|------|------|----------|
| **데이터 부족** | 약관 문서 자체가 없음 | 보험사에 문의 |
| **확인 불가** | 문서는 있지만 해석 불가 | 약관 직접 확인 또는 상담 |

Q5는 **"확인 불가"** 상태입니다.

## 자주 묻는 질문 (FAQ)

### Q. 왜 다른 비교 사이트는 이 정보를 제공하나요?
A. 다른 사이트는 다음 중 하나일 가능성이 높습니다:
1. **추정값 사용** (명시하지 않음)
2. **일부 보험사만 제공** (전체 비교 아님)
3. **수동 입력 데이터** (업데이트 지연/오류 가능)

본 서비스는 **자동화 + 근거 기반**을 원칙으로 하여,
확인할 수 없는 정보는 제공하지 않습니다.

### Q. 언제쯤 이 기능이 추가되나요?
A. Step3 개선 작업이 완료되면 (예상: 2-3일 소요),
Q5를 포함한 여러 질문에 대한 답변이 가능해집니다.

개선 완료 시 별도 공지 예정입니다.

### Q. 다른 질문은 왜 답변 가능한가요?
A. Q1 (가성비 Top3), Q12 (삼성 vs 메리츠), Q14 (보험료 Top4)는
**보험료 데이터**를 사용하며, 이는 API로 확보한 확실한 데이터입니다.

Q3 (의무담보), Q5 (감액/면책), Q11 (한도)은 **약관 문서**에서
추출해야 하는 정보로, 현재 기술적 한계가 있습니다.

- **Q3**: 상품 구조 분석 기능 미구현 (주계약/특약 구분 불가)
- **Q5**: 담보 귀속 확인 로직 불충분 (조건이 어느 담보의 것인지 불명확)
- **Q11**: 데이터 구조 불일치 (일당 금액과 기간 한도가 구분되지 않음)
