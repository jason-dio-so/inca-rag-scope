4가지 예제를 준비했으며, 오늘은 예제 3번 '상품 종합 비교'를

**화면**:
- 첫 화면 (시스템 안내 + 4개 예제 버튼)
- 예제 3번 버튼이 이미 활성화되어 있음 (기본 로드)

### [30초] 예제 3 - 상단 확인 (1분)

**말하기**:
```
"화면 상단을 보시면, 사용자 질문
'삼성생명과 한화생명 건강보험을 비교해줘'가 표시되고,

바로 아래 '비교 대상 상품' 섹션에서
- 삼성생명 건강보험
- 한화생명 건강보험
두 상품이 명확히 표시됩니다.

회사명과 상품명을 항상 함께 보여주는 것이 핵심입니다."
```

**화면 포인트**:
- ✅ 질문 재확인 (User Message Bubble)
- ✅ 📋 비교 대상 상품 섹션
- ✅ 회사명 + 상품명 병기

**고객 확인 사항**:
- [ ] 회사명 + 상품명 표시 위치가 적절한가?
- [ ] 폰트 크기 / 강조 정도가 적절한가?

### [1분 30초] 예제 3 - 9개 담보 비교표 (1분 30초)

**말하기**:
```
"아래로 내려가면 '보장 비교표'가 있습니다.
9개 담보가 테이블로 정리되어 있고,
각 담보마다 두 회사의 값이 나란히 비교됩니다.

예를 들어 첫 번째 담보인 '암 진단비'를 보면,
- 삼성생명: 3,000만원 (일반암)
- 한화생명: 5,000만원 (일반암)

각 값 아래에는 📄 아이콘이 있고, 이건 문서 출처입니다.
클릭하면... [클릭]"
```

**화면 포인트**:
- ✅ 테이블 스크롤 (9개 행 모두 보이는지)
- ✅ 각 셀: 값 + Evidence (📄 아이콘)
- ✅ 📄 클릭 → 모달 팝업

**모달 시연** (30초):
```
"이렇게 Evidence 상세 모달이 뜹니다.
- 담보명과 회사명
- 값
- 문서 출처 (가입설계서 p.3, 약관 p.12)
- 그리고 여기 Snippet 영역에는 실제 구현 시 원문 발췌가 들어갑니다.

현재는 프로토타입이므로 placeholder입니다.

'원문 보기' 버튼도 실제로는 PDF 뷰어로 연결될 예정이지만,
지금은 클릭해도 placeholder 알림만 뜹니다."

[닫기 버튼 클릭]
```

**고객 확인 사항**:
- [ ] 9개 담보가 모두 필요한가? (일부만 보여줄지)
- [ ] 테이블이 너무 길면 접기 기능이 필요한가?
- [ ] Evidence 모달에 필요한 정보가 충분한가?
- [ ] "원문 보기" PDF 연결이 필수인가, 나중에 해도 되는가?

### [3분] 예제 3 - Notes 상세 (1분)

**말하기**:
```
"테이블 아래로 내려가면 '📝 요약' 섹션이 있습니다.

'두 상품 모두 암, 뇌졸중, 심장질환...'으로 시작하는 요약 문장이 있고,
'상세 내용 펼치기'를 클릭하면... [클릭]

이렇게 7개 항목의 상세 비교가 나타납니다.
예를 들어,
- '암 진단비: 한화생명이 5,000만원으로 더 높습니다 (삼성생명 3,000만원).'

여기서 주의할 점은, 우리는 '더 높습니다'처럼 사실만 말하고,
'유리합니다' '추천합니다' 같은 판단은 절대 하지 않습니다."
```

**화면 포인트**:
- ✅ Notes summary (항상 표시)
- ✅ "상세 내용 펼치기" 클릭 → 7개 bullet point
- ✅ 사실 비교만 (추천/판단 표현 없음)

**고객 확인 사항**:
- [ ] Notes 상세 7개 항목이 너무 많은가? (3~5개로 줄일지)
- [ ] 기본 상태를 "펼쳐진 상태"로 할지, "접힌 상태"로 할지?

---

### [4분] 예제 3 - 주의사항 (30초)

**말하기**:
```
"마지막으로 '⚠️ 주의사항' 섹션입니다.
펼치기를 클릭하면... [클릭]

5개 항목이 나오며, 시스템의 한계를 명확히 밝힙니다.
- 본 비교는 약관 및 가입설계서에 기반한 정보 제공입니다.
- 개인 조건에 따른 보험료 계산은 포함되지 않습니다.
- 상품 추천이나 가입 판단을 대신하지 않습니다.

이 섹션은 항상 표시되며, 법적 책임 범위를 명확히 합니다."
```

**화면 포인트**:
- ✅ Limitations (기본 접힌 상태)
- ✅ 5개 한계 명시

**고객 확인 사항**:
- [ ] 주의사항이 너무 보수적인가? (문구 조정 필요 여부)
- [ ] 기본 상태를 "펼쳐진 상태"로 강제할지?

### [4분 30초] 예제 1 - 보험료 경고 (1분)

**말하기**:
```
"이제 예제 1번 '보험료 비교'를 짧게 보겠습니다.
[예제 1 버튼 클릭]

화면 상단에 노란색 경고 배너가 강제로 표시됩니다.

'⚠️ 보험료 안내
보험료는 개인별 조건 (나이, 성별, 건강상태 등)에 따라 달라집니다.
본 시스템은 개인별 보험료 계산 기능을 제공하지 않으며,
문서상 예시만 표시합니다.'

테이블을 보시면 '월 보험료 (참고)'라는 라벨이 있고,
값에도 '약 50,000원 (예시)'처럼 명확히 표기합니다.

아래 주의사항에도 보험료 관련 경고 2개가 최상단에 배치됩니다."
```

**화면 포인트**:
- ✅ Premium notice 배너 (노란색)
- ✅ "(참고)" 라벨
- ✅ "(예시)" 표기
- ✅ Limitations 최상단 경고 2개

**고객 확인 사항**:
- [ ] 보험료 경고가 충분한가? (더 강하게 표시할지)
- [ ] 현재 표기 방식 (예시 금액 + 조건)이 적절한가?
- [ ] 아예 보험료 비교 기능을 제거할지?

## 예상 질문 & 답변

### Q1: "실제 LLM이 답변하는 건가요?"

**A**: 아니요, 현재는 고정된 JSON 데이터를 렌더링하는 프로토타입입니다.
실제 구현 시 LLM이 이 JSON 구조를 생성하게 됩니다.

### Q2: "다른 회사도 추가할 수 있나요?"

**A**: 네, 신규 보험사 추가는 JSON 데이터만 준비하면 자동으로 렌더링됩니다.
UI 코드 변경은 불필요합니다.

### Q3: "보험료를 정확히 계산할 수 없나요?"

**A**: 현재 시스템 범위는 '문서 기반 정보 비교'이며,
개인별 보험료 계산은 별도 시스템이 필요합니다.
고객 요구사항에 따라 향후 연동 검토 가능합니다.

### Q4: "원문 PDF를 바로 볼 수 있으면 좋겠어요."

**A**: 네, 'DECISIONS_TO_LOCK.md'에 우선순위 결정 항목으로 포함되어 있습니다.
필수로 하실지, 다음 단계로 미룰지 결정 부탁드립니다.
